# Changelog

## [1.1.0] - 2026-03-01

### ✨ 核心新功能：Hosts 文件管理

1.1.0 版本将 iEnvs 从单一的环境变量管理工具升级为**环境变量 + Hosts 双重管理工具**。

#### Hosts 管理能力
- **Hosts 分组管理**：创建、编辑、删除 hosts 分组，支持分组启用/禁用
- **Hosts 条目管理**：添加、编辑、删除 hosts 条目（IP 地址、主机名、注释）
- **冲突检测**：自动检测 IP 地址或主机名冲突
- **导入/导出**：支持 JSON 格式备份 hosts 数据
- **可视化界面**：独立的 Hosts 侧边栏、详情页、空状态页面
- **状态栏集成**：菜单栏可直接切换 hosts 分组
- **自动同步**：启用分组自动写入 `/etc/hosts`，禁用自动移除

#### 技术实现
- `HostsFileManager`：Hosts 文件读写管理，支持标记区域管理
- `HostsGroupViewModel`：Hosts 分组管理的 MVVM 实现
- `HostsConflictDetector`：IP 和主机名冲突检测
- `HostsValidators`：IP 地址和主机名格式验证

### 📎 附加功能：现有配置导入管理

智能识别并导入已有的配置到 iEnvs 管理：

- **环境变量现有配置检测**：自动扫描 `~/.zshrc` 或 `~/.bashrc` 中的非 iEnvs 管理 export 语句
- **Hosts 现有配置检测**：自动扫描 `/etc/hosts` 中的非 iEnvs 管理条目
- **一键迁移**：支持将现有配置迁移到 iEnvs 管理，迁移时可修改值
- **直接删除**：支持直接从原配置文件删除配置，无需迁移
- **实时展示**：工具栏显示检测到的未管理配置数量提示

### 🔧 其他更新

#### 版本管理
- 新增全局版本管理脚本 `scripts/version-manager.sh`
- 统一版本号管理：代码、project.yml、CHANGELOG 保持一致

#### 安全机制
- 所有文件修改前自动备份
- 删除操作需二次确认
- 迁移时明确提示将从原配置文件删除

### 🌐 本地化
- 1.0.1 版本已提供完整中英文双语支持
- 1.1.0 新增界面元素已补充本地化

## [1.0.0] - 2026-02-10

### ✨ 初始版本

#### 核心功能
- 环境变量分组管理：创建、编辑、删除分组
- 环境变量管理：添加、编辑、删除、批量导入

- MVVM 架构
